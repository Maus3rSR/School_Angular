---
name: detail-demo-formateur
description: Définir les démos formateur
---

- Pour chaque concept, détailler une démo formateur :
  - Point de départ du code (état du projet, fichiers concernés).
  - Une démo par sous-concept (pas de sous-concept sans démo).
  - Étapes successives de la modification.
  - Dossiers/fichiers à créer explicitement.
  - Rappels de code concrets à fournir (imports, signatures, extraits de template et service).
  - Résultat observable dans le navigateur (ex : nouvelle liste, page de détail fonctionnelle).
- Mettre en avant le raisonnement (pourquoi on crée un service, pourquoi on factorise en composant).
- Mettre les démos formateur dans un dossier `demos/` dans `docs/`.
