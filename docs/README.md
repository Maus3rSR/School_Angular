# Support pédagogique WishFlix

Ce dossier contient le support des 5 séances (3h30 chacune) pour construire le mini Netflix "WishFlix" pas à pas : un catalogue de jeux vidéo inspiré de Netflix.

## Progression globale

1. **`seance-1.md`** - Fondations Angular : Signals, control flow moderne (`@if`, `@for`), data binding
2. **`seance-2.md`** - Composants et communication : `input()`, `output()`, signals dérivés, gestion des favoris
3. **`seance-3.md`** - Services et HTTP : HttpClient, Observables, `toSignal()`, consommation d'API REST
4. **`seance-4.md`** - Routing et navigation : routes paramétrées, guards, lazy loading, page 404
5. **`seance-5.md`** - Formulaires et authentification : Reactive Forms, login/logout, HTTP Interceptors

## Objectif final

À l'issue des 5 séances, les étudiants seront capables de :

- ✅ Construire une SPA Angular moderne avec standalone components
- ✅ Consommer une API REST (préparation à l'intégration d'une API Java)
- ✅ Gérer l'authentification et protéger des routes
- ✅ Découper une UI complexe en composants réutilisables
- ✅ Créer des formulaires avec validation

## Comment utiliser ce support

- **Côté enseignant** : suivre les démos dans l'ordre, chaque séance s'appuie sur la précédente
- **Côté étudiant** : réaliser les exercices "version 2" après chaque démo
- **Côté révision** : utiliser les sections d'auto-évaluation en fin de séance

## Important

- Le projet Angular est **déjà initialisé** avec tout le code "placeholder" nécessaire
- Les étudiants complètent progressivement les TODO sans partir de zéro
- Aucune solution complète n'est fournie : l'apprentissage se fait par la pratique guidée
- Les images de jeux vidéo proviennent de https://dev.me/products/image-placeholder (catégorie "Games")
